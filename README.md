# AI, Machine Learning and Data Mining at Hochschule Bochum

Welcome to the GitHub organization of the **Applied Computer Science Group** at
**Bochum University of Applied Sciences**.

The group is led by **Jörg Frochte**, Professor of Applied Computer Science at
Hochschule Bochum / Bochum University of Applied Sciences, and is primarily based
at the Velbert/Heiligenhaus campus.

We work at the intersection of applied artificial intelligence, machine learning,
data mining, learning analytics, explainable AI, sustainable AI, and
interdisciplinary data science.

This GitHub organization hosts selected public research software, reproducibility
artifacts, prototypes, teaching-related code, and supporting material developed
in the context of our research and teaching activities.

## Research focus

Our work connects machine learning methodology with real-world applicability.
We focus on applied and trustworthy AI systems that are useful, understandable,
resource-aware, and reproducible in practical settings.

Typical research areas include:

- **Applied AI, Machine Learning and Data Mining**  
  Methods for extracting robust, interpretable, and actionable knowledge from
  data, with applications in engineering, industry, education, robotics, and
  interdisciplinary data science.

- **Trustworthy and Explainable AI**  
  Models and methods that make AI-based decisions more transparent,
  interpretable, and accountable, especially in applied decision-support
  scenarios.

- **Continual, Resource-Efficient and Sustainable Learning**  
  Learning systems that adapt to changing requirements without being retrained
  from scratch, with attention to computational efficiency, data protection,
  maintainability, and sustainable deployment.

- **Educational Data Mining and Learning Analytics**  
  Data-driven methods to better understand learning processes, support student
  success, and develop trustworthy educational technologies.

We are particularly interested in challenging applied machine learning problems
where standard approaches are insufficient, for example because data is scarce,
problem formulations are ill-posed, domain knowledge is essential, or models
need to operate under practical constraints.

## What you can find here

Repositories in this organization may include:

- research software accompanying publications,
- reproducibility artifacts and experimental code,

- datasets or dataset utilities where publication is legally and ethically

reproducibility, reuse, citation, and collaboration.

## Open science and licensing

We support open science and reproducible research. Whenever the legal framework,
project agreements, data protection requirements, and cooperation contracts allow
it, we aim to release code, data, models, and documentation publicly.

Our open-source releases typically use permissive licenses such as **MIT**,
**BSD**, or **Apache 2.0**, especially when this simplifies reuse by research
partners, industry partners, and the wider community. The appropriate license is
chosen individually for each project.

Please check the `LICENSE`, `CITATION.cff`, and repository-specific README files
before reusing code or data.

## For contributors

Contributions are welcome where repositories are explicitly open for external
collaboration.

Before contributing, please read the repository-specific contribution guidelines.
In general, contributions should:

- be compatible with the repository license,
- avoid including personal, confidential, or third-party data without permission,
- avoid committing credentials, tokens, API keys, or other secrets,
- include tests or examples where appropriate,
- document relevant assumptions and dependencies,
- cite related publications or datasets where applicable.

For project-specific questions, please use the issue tracker of the corresponding
repository if it is enabled.
